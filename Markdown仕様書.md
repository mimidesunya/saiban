# 裁判文書用Markdown仕様書

本システムでは、標準的なMarkdownをベースに、裁判文書特有の書式（階層構造、右寄せ・左寄せ、表組みなど）を自動変換するための拡張仕様を採用しています。

## 1. 基本書式

### 見出し
- `# タイトル`: 文書中央に配置される大見出し（`<h1>`）となります。
- `## 第1 見出し`: 階層構造を持つ見出し。Markdownプレビューでの見栄えを考慮し、`##` を推奨します。

### 段落
- 通常のテキストは、自動的に1文字インデント（字下げ）され、両端揃えで配置されます。

---

## 2. 階層構造（リーガルナンバリング）

行頭に特定のマーカーを置くことで、自動的にインデントと番号振りが適用されます。Markdownプレビューで見やすくするために、先頭に `##` を付けることができます。

| レベル | マーカー例 | 変換後の形式 |
| :--- | :--- | :--- |
| 1 | `第1` | 第1、第2... |
| 2 | `1` | 1、2... |
| 3 | `(1)` | (1)、(2)... |
| 4 | `ア` | ア、イ... |
| 5 | `(ア)` | (ア)、(イ)... |
| 6 | `a` | a、b... |
| 7 | `(a)` | (a)、(b)... |

**記述例:**
```markdown
## 第1 請求の趣旨
## 1 被告は、原告に対し...
## (1) 損害の内訳
```

---

## 3. 配置ブロック

特定の範囲を右寄せ、または左寄せにします。

### 右寄せブロック（署名・当事者欄など）
`### --右` で開始し、`### --` で終了します。
- ブロック内の各行は、文書全体の「表組み（リスト形式）」の合計幅に合わせて右側に配置されます。これにより、署名欄と事件情報などの幅が統一されます。

```markdown
### --右
〒100-0001
東京都千代田区...
原告　甲野　太郎
### --
```

### 左寄せブロック（事件番号・宛先など）
`### --左` で開始し、`### --` で終了します。
```markdown
### --左
令和○年（ワ）第○○○号
- 被告：乙山　次郎
### --
```

---

## 4. 表組み（2つのスタイル）

裁判文書の「附属書類」や「事件情報」などに使用します。

### リスト形式（推奨）
Markdownプレビューでも内容が確認しやすく、記述も簡潔です。
`- 項目名：内容` の形式で記述します。
- **グローバル幅調整**: 文書内のすべてのリスト形式テーブル（「附属書類」以外）は、最も長い行に合わせて列幅が自動的に統一されます。

```markdown
- 訴訟物の価格：1,600,000円
- 貼用印紙額：19,500円
```

### パイプ形式
標準的なMarkdownのテーブル記法です。
```markdown
|項目名|内容|
```

### 特殊な処理
- **金額の右寄せ**: 2列目の内容が数字や「円」のみの場合、自動的に右寄せされます。
- **附属書類**: 直前の見出しが「附属書類」の場合、自動的に番号振りが適用される専用のスタイルになります。

---

## 5. 特殊マーカー

### 日付の自動右寄せ
和暦（令和・平成など）または西暦の形式で1行記述すると、自動的に右寄せされます。
- 例: `令和7年1月1日`

### 宛先の自動左寄せ
行末が「御中」または「様」で終わる場合、自動的に左寄せされます。
- 例: `○○地方裁判所 御中`

### 以上（結語）
`以上` とのみ記述された行は、自動的に右寄せされます。

### 改ページ
`### -- 任意のテキスト --` の形式で記述すると、その位置で改ページが挿入されます。
- 例: `### -- 別紙 --`
